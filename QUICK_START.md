# 快速開始：Git 提交優化

## 功能說明
快速解決 Git 提交時的 token 消耗和時間成本問題
創建日期：2025-12-12
創建人：Daniel Chung
最後修改日期：2025-12-12

## 一鍵解決方案

### 日常開發（推薦，節省 70% 時間）

```bash
# 快速提交，只格式化，跳過 mypy
./scripts/quick_commit.sh -f "你的提交訊息"
```

### 提交前檢查（可選）

```bash
# 先檢查，再提交
./scripts/pre_commit_check.sh
./scripts/quick_commit.sh -f "你的提交訊息"
```

## 三種模式

1. **快速模式**（`-f`）：只格式化，跳過 mypy，節省 70% 時間
2. **完整模式**（默認）：運行所有檢查，確保代碼質量
3. **跳過模式**（`-s`）：緊急情況使用，跳過所有檢查

## 性能提升

- ✅ 提交速度：**10-30 秒**（快速模式）vs **5-10 分鐘**（優化前）
- ✅ Token 消耗：**減少 60-90%**
- ✅ 只檢查修改的文件，不檢查整個項目
