# WBS 1.4 ArangoDB 部署子計劃

## 1. 背景與目標

- 為後續知識圖譜與關係查詢需求建立 ArangoDB 集群，於 5 個工作日內完成部署、SDK 與圖模型設計。

## 2. 範圍

- 部署單節點至雙節點模式、備援策略與監控。
- SDK 封裝（Graph API、AQL 查詢、權限管理）。
- 知識圖譜 schema 與圖查詢範例。

## 3. 工作拆解

| 任務編號 | 描述 | 負責人 | 工期 | 依賴 |
|---------|------|--------|------|------|
| 1.4.1 | ArangoDB 安裝、配置、備援策略 | DevOps-1 | 1 天 | 1.0.* |
| 1.4.2 | SDK 封裝（Graph、文檔、AQL 模組） | Backend-1 | 1.5 天 | 1.4.1 |
| 1.4.3 | 知識圖譜建模（entity/edge/schema） | Backend-1 | 1 天 | 1.4.2 |
| 1.4.4 | 圖查詢功能與單元測試 | Backend-1 | 1 天 | 1.4.3 |

## 4. 時間表（2025/12/01 - 2025/12/05）

- 12/01：完成 1.4.1。
- 12/03：完成 1.4.2 SDK。
- 12/04：完成 1.4.3 圖模型設計。
- 12/05：完成 1.4.4 查詢實作與測試。

## 5. 資源與交付物

- 資源：DevOps-1（部署）、Backend-1（SDK/模型）、AI-2（提供圖譜樣本資料）。
- 交付物：
  - `infra/arangodb/` 部署說明、監控設定。
  - `libs/arangodb_client` SDK、AQL 模板。
  - 知識圖譜 schema 文檔、示例查詢腳本。

## 6. 風險與對策

- **圖 schema 變動頻繁**：採版本化 schema，並提供 migration 腳本。
- **查詢效能問題**：事先定義 index 策略並納入性能測試。

## 7. 驗收標準

- 部署環境可啟動並通過健康檢查，failover 測試成功。
- SDK 可執行 CRUD、圖 traversal、AQL 查詢並通過單元測試。
- 提供至少 3 個圖查詢範例（路徑、子圖、屬性過濾）。
