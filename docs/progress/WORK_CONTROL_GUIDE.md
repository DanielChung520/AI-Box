# 工作管制表使用指南

**創建日期**: 2025-01-27  
**創建人**: Daniel Chung  
**最後修改日期**: 2025-01-27

---

## 概述

工作管制表 (`PROJECT_CONTROL_TABLE.md`) 是項目管理的核心文檔，用於追蹤所有階段的工作進度、里程碑狀態和風險。

---

## 文件位置

- **工作管制表**: `docs/PROJECT_CONTROL_TABLE.md`
- **更新腳本**: `scripts/update_project_control.sh`
- **週報生成腳本**: `scripts/generate_weekly_summary.sh`

---

## 更新頻率

### 每日更新

每日更新當前階段的工作項狀態：

```bash
./scripts/update_project_control.sh
# 選擇選項 1: 更新工作項狀態
```

### 每週更新

每週五更新整體進度和生成週報：

```bash
# 更新階段進度
./scripts/update_project_control.sh
# 選擇選項 2: 更新階段進度

# 生成週報摘要
./scripts/generate_weekly_summary.sh
```

### 里程碑更新

里程碑達成時立即更新：

```bash
./scripts/update_project_control.sh
# 選擇選項 3: 更新里程碑狀態
```

---

## 更新內容

### 1. 工作項狀態更新

更新以下信息：
- **狀態**: completed / in_progress / pending / blocked
- **完成度**: 0-100%
- **實際完成日期**: YYYY-MM-DD

### 2. 階段進度更新

更新以下信息：
- **階段狀態**: 已完成 / 進行中 / 未開始 / 延遲 / 阻塞
- **進度百分比**: 0-100%
- **實際完成日期**: YYYY-MM-DD（如已完成）

### 3. 里程碑狀態更新

更新以下信息：
- **狀態**: completed / in_progress / pending
- **實際完成日期**: YYYY-MM-DD
- **延遲天數**: 如延遲

### 4. 風險追蹤更新

添加或更新風險：
- **風險描述**
- **影響**: 高 / 中 / 低
- **可能性**: 高 / 中 / 低
- **狀態**: 已緩解 / 監控中 / 待處理
- **緩解措施**

---

## 狀態標記說明

### 工作項/階段狀態

- ✅ **completed**: 已完成
- 🔄 **in_progress**: 進行中
- ⏸️ **pending**: 未開始
- ⚠️ **delayed**: 延遲
- ❌ **blocked**: 阻塞

### 里程碑狀態

- ✅ **completed**: 已完成
- 🔄 **in_progress**: 進行中
- ⏸️ **pending**: 未開始

---

## 進度計算方法

### 工作項進度

```
工作項進度 = (已完成任務數 / 總任務數) × 100%
```

### 階段進度

```
階段進度 = (已完成工作項數 / 總工作項數) × 100%
```

### 整體進度

```
整體進度 = Σ(階段進度 × 階段權重) / 總權重
```
