# MM-Agent 意圖分類場景庫

## 設計原則

| 原則 | 說明 |
|------|------|
| **去數據化** | 不含具體料號、倉庫代碼、供應商代碼 |
| **語義優先** | 聚焦「查詢」、「統計」、「比較」等意圖表達 |
| **唯一性** | 每個場景表達必須與其他場景有明顯區別 |
| **可擴展** | 預留空間給未來新增場景 |

---

## 意圖分類體系

### 架構圖

```
┌─────────────────────────────────────────────────────────────┐
│                    第一層：AI-Box Router                     │
├─────────────────────────────────────────────────────────────┤
│  GREETING │ THANKS │ COMPLAIN │ CANCEL │ CONFIRM │ ...  │
│    (問候)   (感謝)   (道歉)    (取消)    (確認)           │
└─────────────────────────────────────────────────────────────┘
                            ↓ 轉發
┌─────────────────────────────────────────────────────────────┐
│                    第二層：BPA (MM-Agent)                   │
├─────────────────────────────────────────────────────────────┤
│  KNOWLEDGE_QUERY │ SIMPLE_QUERY │ COMPLEX_TASK │ ...      │
│      (知識查詢)      (數據查詢)      (複雜任務)              │
└─────────────────────────────────────────────────────────────┘
```

### 第一層：AI-Box Router 意圖

| 意圖代碼 | 意圖名稱 | 說明 |
|----------|----------|------|
| GREETING | 問候 | 打招呼、問候 |
| THANKS | 感謝 | 感謝回覆 |
| COMPLAIN | 道歉 | 道歉、抱歉 |
| CANCEL | 取消 | 取消任務 |
| CONTINUE | 繼續 | 繼續執行 |
| MODIFY | 修改 | 重新處理 |
| HISTORY | 歷史 | 顯示歷史記錄 |
| EXPORT | 導出 | 導出數據 |
| CONFIRM | 確認 | 請求確認 |
| FEEDBACK | 反饋 | 記錄反饋 |
| CLARIFICATION | 澄清 | 請求澄清 |
| BUSINESS | 業務 | 業務請求（轉發第二層）← **最大類** |

### 第二層：BPA (MM-Agent) 意圖

| 意圖代碼 | 意圖名稱 | 說明 | 處理目標 |
|----------|----------|------|----------|
| KNOWLEDGE_QUERY | 知識查詢 | 業務知識、專業術語、流程詢問 | → KA-Agent |
| SIMPLE_QUERY | 數據查詢 | 庫存、料號、採購、銷售等簡單查詢 | → Data-Agent |
| COMPLEX_TASK | 複雜任務 | 比較、分析、預測、報告生成 | → ReAct Planner |
| CLARIFICATION | 澄清需求 | 資訊不足、模糊表達 | → 請求用戶補充 |
| CONTINUE_WORKFLOW | 繼續工作流 | 繼續執行工作流 | → 執行中工作流 |

### 知識來源類型（KNOWLEDGE_QUERY 專用）

| 類型代碼 | 類型名稱 | 說明 |
|----------|----------|------|
| INTERNAL | 內部知識 | 公司內部流程、知識庫文件 |
| EXTERNAL | 外部知識 | 專業術語、行業標準、理論 |
| UNKNOWN | 未知 | 無法判斷來源 |

### 閾值策略

| 分數區間 | 策略 |
|----------|------|
| > 0.90 | 直接採用檢索結果 |
| 0.70 - 0.90 | LLM 參考檢索結果判決 |
| < 0.70 | LLM 獨立判斷 |

## 擴展指南

### 新增場景原則

1. **去數據化**：不含具體料號、倉庫代碼
2. **語義獨立**：表達方式獨特，不與現有混淆
3. **均勻分佈**：每個意圖大類不超過總量 30%
4. **定期審視**：每季檢索命中率，低於 70% 需優化

### 數量上限

| 階段 | 建議數量 |
|------|----------|
| MVP | 50-80 |
| 正式版 | 100-150 |
| 上限 | 不超過 200 |
