## 1. 概述

### 1.1 目的

時間分詞器（TimePeriodRAG）是一個基於向量檢索的時間表達識別工具，旨在解決自然語言查詢中時間表達方式多樣性的問題。傳統的正則表達式方案難以覆蓋各種時間表達變體（如「2025年度」、「最近一周」、「Q1 2025」等），本工具通過 RAG 架構實現更靈活、更準確的時間表達解析。

### 1.2 問題背景

在 AI-Box 系統的 Data-Agent 中，IntentAnalyzer 使用正則表達式識別時間：

```python
# 現有實現的局限性
year_month = re.search(r"(\d{4})[年/\-](\d{1,2})", query)
```

**問題**：
- 「2025年度」無法識別（正則只匹配「年」後面直接跟數字）
- 「最近一周」需要額外添加規則
- 中英文混合表達支持困難
- 維護成本高，每新增表達方式需修改代碼

### 1.3 解決方案

通過向量相似度匹配，將時間表達的識別從「規則匹配」轉變為「語義匹配」：

```
用戶輸入 → 向量化 → Qdrant 相似度搜索 → 結構化輸出
```

## 3. 功能說明

### 3.1 模板庫結構

時間表達模板按類別組織，目前支持以下類別：

| 類別 | 粒度 | 示例表達 |
|------|------|----------|
| `year` | YEAR | 「2025年」、「今年」、「last year」 |
| `month` | MONTH | 「1月」、「January」、「上個月」 |
| `year_month` | YEAR_MONTH | 「2025年1月」、「Jan 2025」 |
| `date_range` | RANGE | 「最近7天」、「Q1 2025」、「from...to...」 |
| `decade` | DECADE | 「2020年代」、「21世紀」 |

### 3.2 主要功能

#### 3.2.1 構建索引 (`build_index`)

首次使用時，將模板庫中的所有時間表達向量化並存入 Qdrant：

```python
tool = TimePeriodRAGTool()
result = tool.build_index()

# 輸出範例：
# {
#   "status": "completed",
#   "total_templates": 150,
#   "success_count": 148,
#   "failed_count": 2,
#   "failed_expressions": [...]
# }
```

#### 3.2.2 查詢解析 (`execute`)

將時間表達解析為結構化參數：

```python
result = await tool.execute(
    TimePeriodRAGInput(
        time_expression="2025年度庫存",
        current_year=2026
    )
)

# 輸出範例：
# TimePeriodRAGOutput(
#     original_expression="2025年度庫存",
#     matched_expression="2025年度",
#     category="year",
#     granularity="YEAR",
#     parsed={"year": "2025"},
#     confidence=0.92,
#     start_date="2025-01-01T00:00:00",
#     end_date="2025-12-31T23:59:59"
# )
```

### 3.3 支持的時間表達

#### 3.3.1 中文表達

| 類別 | 示例 | 解析結果 |
|------|------|----------|
| 年份 | 2025年、2025年度 | `{year: "2025"}` |
| 相對 | 今年、去年、明年 | 根據當前年份計算 |
| 月份 | 1月、01月份 | `{month: "01"}` |
| 相對月 | 上個月、下個月 | 根據當前月份計算 |
| 範圍 | 最近7天、最近一周 | `{relative: "last_7_days"}` |
| 季 | Q1 2025、第一季度 | `{year: "2025", quarter: "Q1"}` |
| 半年 | 上半年、下半年 | `{half: "first/second"}` |
| 年初年末 | 年初、年末 | `{relative: "year_start/end"}` |

#### 3.3.2 英文表達

| 類別 | 示例 | 解析結果 |
|------|------|----------|
| 年份 | in 2025、year 2025、CY2025 | `{year: "2025"}` |
| 相對 | this year、last year、next year | 根據當前年份計算 |
| 月份 | January、Jan、month 1 | `{month: "01"}` |
| 範圍 | last 7 days、past week | `{relative: "last_7_days"}` |
| 季 | Q1 2025、2025 Q1 | `{year: "2025", quarter: "Q1"}` |

## 4. 使用方式

### 4.1 基礎使用

```python
from tools.time.time_period_rag import TimePeriodRAGTool, TimePeriodRAGInput

# 獲取工具實例
tool = TimePeriodRAGTool()

# 首次使用時構建索引
tool.build_index()

# 執行時間解析
result = await tool.execute(
    TimePeriodRAGInput(
        time_expression="查詢2025年的庫存統計",
        current_year=2026
    )
)

print(result.start_date)  # 2025-01-01T00:00:00
print(result.end_date)   # 2025-12-31T23:59:59
print(result.granularity)  # YEAR
```

### 4.2 與 IntentAnalyzer 集成

```python
from tools.time.time_period_rag import TimePeriodRAGTool, TimePeriodRAGInput

class IntentAnalyzer:
    def __init__(self):
        self.time_rag = TimePeriodRAGTool()
        # 首次使用時構建索引
        self.time_rag.build_index()

    def _recognize_time_filter(self, query: str, result: IntentAnalysisResult):
        # 提取查詢中的時間表達
        time_match = re.search(r"(\d{4})年|今年|去年|最近|\w+天", query)
        if time_match:
            time_expr = time_match.group(0)
            
            # 使用 TimePeriodRAG 解析
            time_result = await self.time_rag.execute(
                TimePeriodRAGInput(time_expression=time_expr)
            )
            
            result.has_time_filter = True
            result.time_granularity = time_result.granularity
            result.time_start = time_result.start_date
            result.time_end = time_result.end_date
```

## 5. 配置說明

### 5.1 配置常量

| 常量 | 默認值 | 說明 |
|------|--------|------|
| `COLLECTION_NAME` | `time_period_templates` | Qdrant collection 名稱 |
| `EMBEDDING_MODEL` | `nomic-embed-text:latest` | 嵌入模型名稱 |
| `EMBEDDING_URL` | `http://localhost:11434/api/embeddings` | Ollama API 地址 |
| `VECTOR_DIMENSION` | `768` | 向量維度 |
| `SIMILARITY_THRESHOLD` | `0.75` | 相似度閾值 |

### 5.2 環境依賴

確保以下服務正在運行：

```bash
# Qdrant
docker run -d -p 6333:6333 qdrant/qdrant

# Ollama
ollama serve
ollama pull nomic-embed-text:latest
```

## 6. 擴展指南

### 6.1 添加新的時間表達

編輯 `tools/time/time_period_templates.json`：

```json
{
  "categories": {
    "year": {
      "templates": [
        {
          "expression": "2027年", 
          "parsed": {"year": "2027"}
        }
      ]
    }
  }
}
```

然後重新調用 `build_index()` 重建索引。

### 6.2 支持新語言

添加對應語言的模板到相應類別：

```json
{
  "expression": "annee 2025", 
  "parsed": {"year": "2025"},
  "language": "fr"
}
```

## 7. 性能考量

### 7.1 響應時間

| 操作 | 預估延遲 |
|------|----------|
| 嵌入生成 | ~50ms（本地 Ollama） |
| Qdrant 搜索 | ~10ms |
| 總延遲 | ~60-100ms |

### 7.2 模板數量

當前模板庫包含約 150 個時間表達，佔用向量存儲約 150KB，搜索性能優秀。

---

## 8. 限制與注意事項

1. **依賴外部服務**：需要 Ollama 和 Qdrant 正常運行
2. **首次構建較慢**：構建索引需要對每個模板生成向量，約 150 個模板需要 7-10 秒
3. **相似度閾值**：設置為 0.75，過低可能匹配到無關表達，過高可能無法匹配
4. **相對時間計算**：相對時間（如「今年」）需要提供 `current_year` 參數
